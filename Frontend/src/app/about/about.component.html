What did Daniel learn from this full stack stack application?

I invested about 300 hours into this project for over a year to deepen my skills as I integrate a front-end project using Angular 19 plugged into a backend Web API using ASP.NET Core 8.0 and other technologies such as SQLServer, Azure DevOps. Also clean architecture concepts. This project solved a problem I had while doing uber eats delivery.

I have three repositories in total in the creation of this project that can be viewed in Azure DevOps.

First repo is titled 'Uber-Eats-Trip-Delivery-Portfolio-Project' in my Azure DevOps is a pdf scrapper written in C# to grab information from 223 pdfs, convert them into text files, and lastly compile them all into a single CSV file. Then I used that csv file to create business analytics using Power BI.

Second repo is titled 'TripInfo' that is a RESTful Web API using .NET 8.0 that uses CRUD operations to manipulate JSON as a business service for async methods that prints to the development console different calculations for analyzing delivery trip data. Endpoints were tested using Postman.

Third repo is titled 'Frontend' and by using the login details, you can see JSON retrieval from seeded data from a SQLServer cloud database. I hosted this entire application using Azure services.

What does Daniel think a good software developer spends their time on?

I think a good developer solves solutions with both minimal technical effort and maximum tact.

Regardless of what role I am in ast an organization, I must deliver business value. There are many wayss to provide that value. Creating user stories that solve actual problems that real users have.

Testing the software to ensure that it behaves as expected to meet requirements.

Deploying continual releases that are in essence, a minimal viable solution. A minimal viable solution is that of which the software developer created a sufficient solution that immediately addresses the bottlenecks that the actual user is facing; opposed to creating features for the sake of producing code that does not get used by users.

What does Daniel aspire for his future employment at a company?

I aspire to work on a team and create software that will be used by many people.

I think people who work together are much more productive to creating and maintaining complex systems.

I look forward to spending years getting to deeply know a code base so that I can inform myself about what good architecture and design looks like.

Because any technology looks great and works well out of the box. But what happens after 4 to 7 years of maintaining that project or building adjacent ones? I want to be there when that happens.
